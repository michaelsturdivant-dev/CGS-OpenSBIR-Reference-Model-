# Contributing

Thank you for considering a contribution to CGS Sentinel OpenSBIR™.

## Contribution Boundary

This project accepts contributions that improve the public reference model, documentation, examples, tests, and safe workflow patterns.

Do not submit:

- Private customer data
- Protected SBIR/STTR technical data
- Proprietary CGS Sentinel AI Engine logic
- Secret credentials or tokens
- Private prompts or internal commercial playbooks

## Contribution Areas

Useful contributions include:

- Documentation improvements
- Example workflows
- Public-safe schemas
- Tests
- Accessibility improvements
- Installation instructions
- Non-sensitive architecture examples

## Review Standard

All contributions should be clear, auditable, minimal, and aligned with the open-core boundary of this project.
