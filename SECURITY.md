# Security Policy

## Public Repository Boundary

This repository is a public reference model. Do not commit secrets, credentials, tokens, private data, customer data, protected SBIR/STTR technical data, or proprietary CGS Sentinel AI Engine logic.

## Never Commit

- Secret credentials
- Private customer or partner data
- Protected technical data
- Local databases
- Virtual environments
- Private prompt chains
- Proprietary kernels or commercial playbooks

## Reporting Issues

Security concerns should be reported privately to Closed Gap Solutions leadership before public disclosure.

## Development Principle

The public repository should remain safe, reproducible, and minimal. Sensitive logic belongs in private repositories or protected deployment environments.
