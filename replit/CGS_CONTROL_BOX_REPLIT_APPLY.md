# Replit Apply File — CGS Control Box

Apply the CGS operating command to the Replit/Sentinel runtime.

## Command

Protect the engine. Display the capabilities. Sell the packages. Route everything through the box. No uncontrolled custom work. No public exposure of proprietary logic. No delivery without review. No partner representation without authority.

## Replit/Sentinel Role

Replit/Sentinel is the engine draft runtime. It classifies intake, validates package code, assigns tier, drafts standard outputs, flags risk, and sends work to CGS review.

## Required Project Fields

- Project ID
- Client or business equity status
- Package code
- Payment or authority status
- Intake status
- Tier
- Gate
- RAG
- Owner
- Due date
- Blocker
- Next action
- Partner authority tag
- Change order status
- Archive link
- Reviewer
- Delivery status

## Hard Stops

- Missing Project ID
- Missing payment or authority
- Missing intake
- Unknown package code
- Missing tier or gate
- Tier 3 without review
- Scope expansion without change order
- Partner without authority
- Public exposure of proprietary logic
- Delivery without archive

## Standard Output

Every engine output must include: BLUF, intake summary, readiness or gap score, recommended path, deliverable body, risk notes, action register, change order flag, partner authority tag, archive record, and upsell path.
