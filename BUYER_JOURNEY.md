# Buyer Journey

## Purpose

This file defines how a visitor moves from GitHub interest to a paid CGS engagement.

## Stage 1: Awareness

The buyer finds the repository through a referral, proposal, GitHub search, LinkedIn post, grant conversation, SBIR discussion, or partner introduction.

Buyer question:

> What is this and why should I care?

Repository answer:

> This is a public reference model for organizing SBIR, OTA, grant, compliance, and execution workflows.

## Stage 2: Trust

The buyer reviews:

- `README.md`
- `SBIR_DATA_RIGHTS.md`
- `COMMERCIALIZATION.md`
- `SECURITY.md`
- `docs/architecture/overview.md`

Buyer question:

> Is this serious, safe, and credible?

Repository answer:

> The public model is transparent, protected boundaries are clear, and commercial implementation is separated from open-source reference code.

## Stage 3: Fit

The buyer compares their problem to the listed use cases:

- SBIR/STTR readiness
- OTA concept development
- Grant operations
- Community infrastructure pilots
- Healthcare access coordination
- Workforce and education programs
- Government contracting support

Buyer question:

> Does this apply to my situation?

Repository answer:

> If the buyer has funding complexity, stakeholder complexity, compliance complexity, or execution complexity, it fits.

## Stage 4: Action

The buyer requests one of these:

- Readiness Review
- OpenSBIR Sprint
- Implementation Package
- Enterprise Deployment

Buyer question:

> What do I do next?

Repository answer:

> Contact Closed Gap Solutions for a readiness sprint, scope review, or implementation package.

## Stage 5: Conversion

CGS conducts:

1. Discovery call
2. Problem intake
3. Opportunity review
4. Fit determination
5. Scope of work
6. Paid engagement

## Conversion Trigger Statements

Use these when speaking with prospects:

- If you already have an opportunity but not a workflow, you need a readiness sprint.
- If you have multiple partners but no operating structure, you need an implementation package.
- If you have deadlines, grants, and stakeholders scattered across emails and folders, you need Sentinel-style operating discipline.
- If you want open-source credibility without giving away your proprietary execution model, this structure fits.
