# CGS Operating Command

Protect the engine.
Display the capabilities.
Sell the packages.
Route everything through the box.
No uncontrolled custom work.
No public exposure of proprietary logic.
No delivery without review.
No partner representation without authority.

## Source Rule

Wix displays the offer.
Stripe collects payment.
Portal captures intake.
Netlify routes.
Replit/Sentinel drafts.
CGS reviews.
Drive archives.
Gmail delivers.
Dashboard governs.
GitHub protects source.
Private Lab protects the engine.

## Authorization Spine

Every project must pass through:

1. Project ID
2. Payment or Authority
3. Intake
4. Package Code
5. Tier
6. Gate
7. Engine Draft
8. CGS Review
9. Archive
10. Delivery
11. Dashboard

## Visibility Rule

Public sees capability.

Operators see workflow.
