# Replit Agent Change Request

## Objective

Update the CGS Replit prototype or MVP interface so it matches the current Closed Gap Solutions public positioning and supports beta testing of the OpenSBIR Readiness Sprint buyer journey.

## Public Positioning

Closed Gap Solutions helps organizations identify operational gaps, structure readiness plans, build implementation systems, and convert ideas into executable programs, proposals, dashboards, and decision-support artifacts.

CGS supports technical founders, small businesses, ministries, nonprofits, workforce programs, and community partners through structured intake, readiness assessments, implementation planning, knowledge management, and AI-supported administrative execution.

## Required Replit Updates

Update visible copy, page titles, navigation labels, form options, and prototype instructions so the app presents CGS as:

- Operational readiness support
- Administrative intelligence support
- Workforce readiness support
- Technical readiness support
- Grant and proposal readiness support
- OpenSBIR Readiness Sprint support
- Dashboard and decision-board support
- Implementation support

## Primary User Paths

The prototype should support these public-safe paths:

1. Buy or request a Readiness Assessment
2. Request an OpenSBIR Readiness Sprint
3. Book a Discovery Call
4. Review the GitHub Reference Model
5. Submit Website Beta Feedback
6. Request Dashboard or Decision Board Support
7. Request Grant or Proposal Readiness Support
8. Request Workforce Readiness Support

## Form Dropdown Options

Use these inquiry options:

- Readiness Assessment
- OpenSBIR Readiness Sprint
- Discovery Call
- Website Beta Feedback
- Grant / Proposal Readiness
- Dashboard / Decision Board Support
- Workforce Readiness Support
- Implementation Support
- General Inquiry

## Architecture Guardrails

- Wix is the public engagement and front-door layer.
- GitHub is the public proof and reference model layer.
- Replit supports MVP and prototype interaction.
- Sentinel is the protected internal execution layer.

Do not expose private execution logic, protected templates, internal prompts, client-specific data, clinical or billing logic, credentialing workflows, or proprietary routing rules.

## Beta Instructions to Surface

Include a beta test page or section that asks testers to verify:

- Whether the CGS offer is clear in under 30 seconds
- Whether OpenSBIR Readiness Sprint is understandable
- Whether each CTA works
- Whether the contact/request path works
- Whether the mobile experience works
- Whether old legacy training language has been removed
- Whether public proof and protected execution boundaries are clear

## CTA Labels

Use these CTA labels consistently:

- Buy Readiness Assessment
- Request Readiness Sprint
- Book Discovery Call
- View GitHub Reference Model
- Submit Beta Feedback

## Success Criteria

- Public copy aligns with CGS operational readiness positioning.
- Beta testers can complete the buyer journey without confusion.
- The prototype supports assessment, sprint request, discovery call, proof review, and beta feedback.
- The app does not expose protected Sentinel logic.
- The app remains suitable for linking from Wix and GitHub.
